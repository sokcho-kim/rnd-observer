# R&D 공모전 Agent 워크플로우

> **문서 버전**: v0.1
> **작성일**: 2026-01-05
> **목적**: provenance-cohort 프로젝트 경험을 바탕으로 R&D 공모전 참여 워크플로우 정리
> **향후 계획**: 별도 레포(rnd-agent)에서 구현 예정

---

## 배경

### 이번 공모전 (AI Co-Scientist Challenge Korea Track2)

| 항목      | 내용                                              |
| --------- | ------------------------------------------------- |
| 발견 경로 | 인스타그램 (비공식)                               |
| 주제 선정 | 현재 업무(급여기준 데이터)에서 자연스럽게 도출    |
| 작업 기간 | 2025.12.25 ~ 2026.01.02 (약 1주)                  |
| 결과      | 사전제안서 v7.0 제출 완료, 예선 발표 대기 (01.19) |

### 느낀 점

- 공고 발견이 우연에 의존 (인스타그램)
- 자료 조사, 제안서 작성 과정이 수동적
- 워크플로우를 정형화하면 다음에 더 효율적

## 워크플로우 설계

### Phase 1: 공고 수집 (Scraping)

#### 공식 루트

| 출처      | URL                         | 비고                                 |
| --------- | --------------------------- | ------------------------------------ |
| IRIS      | https://www.iris.go.kr      | 범부처통합연구지원시스템, 과기부 R&D |
| NTIS      | https://www.ntis.go.kr      | 국가과학기술지식정보서비스           |
| TIPA      | https://www.tipa.or.kr      | 중기부 기술개발                      |
| K-Startup | https://www.k-startup.go.kr | 창업 지원                            |
| aifactory | https://aifactory.space     | AI 특화 경진대회                     |

#### 비공식 루트

- SNS (인스타그램, 링크드인, X)
- 뉴스/기사 (연합뉴스, 전자신문, 데일리팜 등)
- 커뮤니티 (okky 등)

#### 기능 요구사항

```
[입력]
- 공식 사이트 URL 목록
- SNS 키워드 (선택)

[출력]
- 신규 공고 목록 (JSON/MD)
- 알림 (선택)
```

---

### Phase 2: 공고 분석 & 체크리스트 생성

#### 기능 요구사항

```
[입력]
- 공고 URL 또는 PDF

[출력]
- 공고 요약
  - 주최/주관/운영
  - 목적
  - 대상 분야
- 일정 타임라인
  - 신청 기간
  - 심사 일정
  - 수행 기간
  - 결과 발표
- 자격 요건 체크리스트
  - [ ] 참가 자격
  - [ ] 제한 사항
- 제출물 체크리스트
  - [ ] 제안서 (양식)
  - [ ] 발표자료
  - [ ] 기타
- 평가 기준 (배점 포함)
- 지원 사항 (GPU, 예산, 멘토링 등)
```

#### 이번 공모전 예시

```markdown
## AI Co-Scientist Challenge Korea Track2

### 주최/주관
- 주최: 과학기술정보통신부
- 주관: 한국연구재단
- 운영: (주)인공지능팩토리

### 일정
- 신청: 2025.12.10 ~ 2026.01.02
- 예선 발표: 2026.01.19
- 본 개발: 2026.01.19 ~ 04.03
- 본심사: 2026.04

### 평가 기준 (사전심사)
| 항목 | 배점 |
|------|------|
| 파급성 | 30점 |
| 독창성 | 30점 |
| 구체성/실현가능성 | 20점 |
| 검증 방법 타당성 | 20점 |

### 지원 사항 (통과 시)
- GPU 4~8장 (A100급)
- LLM/API 지원
- 팀당 약 3,000만원 상당
```

### Phase 3: 주제 추천 (Agent 기반)

#### v1 (MVP)

- 공고 키워드 추출
- 사용자가 입력한 관심 분야/기술 스택과 매칭
- 간단한 프롬프트 기반 추천

```
[입력]
- 공고 요약 (Phase 2 결과)
- 관심 분야: ["RWE", "NLP", "의료 AI"]
- 보유 기술: ["Python", "LangGraph", "Neo4j"]

[출력]
- 추천 주제 3개
- 각 주제별 적합도 점수
- 근거 설명
```

#### v2 (고도화)

- Git 커밋, 코드베이스 자동 분석
- 현재 진행 중인 프로젝트 파악
- "너 이거 하고 있으니까 이 공모전 맞겠다" 추천
- 보유 데이터/인프라와 공모전 요구사항 매칭

### Phase 4: 자료 조사 (Research Agent)

#### 조사 대상

| 자료 유형 | 출처                                 | 방법                        |
| --------- | ------------------------------------ | --------------------------- |
| 논문      | Google Scholar, arXiv, DBpia, PubMed | 키워드 검색 + 요약          |
| 기사      | 데일리팜, 팜뉴스, 전자신문           | 스크래핑 + 핵심 인용문 추출 |
| 경쟁 환경 | 웹 검색                              | 기존 솔루션, 플랫폼 조사    |
| 시장 규모 | 보고서, 통계청                       | 수치 근거 확보              |

#### 이번 공모전에서 수행한 조사

| 조사 항목         | 결과                                        |
| ----------------- | ------------------------------------------- |
| 글로벌 RWE 플랫폼 | Flatiron, IQVIA, TriNetX 등 6개 분석        |
| 코호트 설계 도구  | OHDSI ATLAS, PheKB 등 4개 분석              |
| 한국 RWE 인프라   | 심평원, 에비드넷/피더넷                     |
| 빅파마 RWE 전략   | 화이자, 암젠, 노바티스                      |
| 핵심 검증         | "급여기준→코호트 자동화" 도구 없음 확인    |
| 논문 분석         | KHIS 용어표준화(2025), EMR 상호운용성(2012) |
| 기사 스크래핑     | 화이자 RWE, 심평원 가이드라인               |
